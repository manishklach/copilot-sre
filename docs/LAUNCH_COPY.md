# Launch Copy

## README hero blurb

Copilot SRE turns GitHub Copilot CLI into an incident commander. It assembles alerts, logs, metrics, deploy history, GitHub activity, runbooks, mitigation options, and rollback guidance into one operational brief, then hands that context to Copilot for faster diagnosis and safer action.

## LinkedIn post

Built a prototype called **Copilot SRE**: an incident-response cockpit on top of GitHub Copilot CLI.

The idea is simple: during an outage, the problem is not asking Copilot a question. The problem is gathering the right context fast enough.

Copilot SRE pulls together:
- telemetry and logs
- deploy history
- GitHub commits and PRs
- runbooks
- mitigation options
- rollback guidance
- a Copilot-ready investigation brief

So instead of starting with a vague prompt, Copilot starts with an operational briefing.

It’s a strong GitHub + Azure + Copilot CLI story, and I think this is the kind of direction that could make terminal agents much more useful in real engineering operations.
